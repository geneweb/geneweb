GeneWeb - Problemen

Hier wordt de oplossing voor twee veel voorkomende problemen met "gwd" uitgelegd.

Je hebt de GeneWeb daemon, "gwd", gestart. Alles lijkt in orde...

Maar dan...

Geen netwerkaansluiting

  • Je surft naar het adres van je eigen GeneWeb server, bijvoorbeeld: 

         http://localhost:2317/jansen
    De browser antwoordt dat er geen netwerkaansluiting is.

    Dit probleem treedt op bij verschillende versies van de Internet Explorer.

  • Wij hebben verschillende oplossinigen gekregen van gebruikers van GeneWeb (bedankt). Probeer dus eens het volgende:

    • Met Internet Explorer kies het menu "View", "Options Internet". In de sectie "Connection" moet je aanduiden: "Do not use a proxy server for local adresses".

    • [Versie 3.02] Met Internet Explorer kies het menu "View", "Options Internet", "Connection". In de instellingen "Proxy server", "Installation" is een lijst met adressen waarvoor de proxy server niet tussenbeide komt. Vemeld daar de locale adressen, zoals: 
           127.0.0.1;localhost;computer

    Als het dan nog niet werkt raden wij u de Netscape browsers aan.(Ze zijn gratis).

Cannot access file "start.txt"

  • Een ander veelvuldig voorkomend probleem is een fout met het bericht: "Cannot access file start.txt".
  • Als u GeneWeb gebruikt onder Windows, hebt u misschien "pkunzip" gebruikt bij de installatie. Het is dan erg belagnrijk om de opties "-e -d" te gebruiken: 
         pkunzip -d -e geneweb.zip
    Als u dit niet gedaan hebt, begin dan terug van vooraf aan, met de juiste opties voor "pkunzip" deze keer.
